1. 算法的定义

1.1 计算问题与算法的定义

计算问题: 给定数据的输入，计算满足某种性质的输出的问题

算法: 将计算问题的输入转换为输出的一系列计算步骤

1.2 算法的性质

  1. 有穷性
  2. 确定性
  3. 可行性

void merge(vector<int> &arr,int start,int end)
{

    // left = arr[start:mid]
    // right = arr[mid+1:end]
    int i = start;

    vector<int> temp;
    while (i <= mid && j <= end)
    {
        if (arr[i] > arr[j])
        {
            temp.push_back(arr[j]);
            j++;
        } else
        {
            temp.push_back(arr[i]);
            i++;
        }
    }
    while (i <= mid)
    {
        temp.push_back(arr[i++]);
    }
    while (j <= end)
    {
        temp.push_back(arr[j++]);
    }
    for (int k = start; k <= end; k++)
    {
        arr[k] = temp[k - start];
    }
}

渐进分析的主要工具是==渐进记号==。渐进记号主要以渐进界的形式表示时间效率。渐进界本身也同样是输入数据规模的函数，只不过更加简洁，且表达式中没有常数系数干扰。假设数据规模足够大的情况下，在忽略常数系数的影响下，渐进符号表示算法的时间效率与渐进界之间的关系。

关系可以分为3种，分别是渐进紧确界(asymptotically tight bound),渐进上界(asymptotically upper bound)和渐进下界(asymptotically lower bound),分别可以用来表示。

    2.1 二分查找

    二分查找(BinarySearch)伪代码如下:

    输入: 长度为n的数组A，数组下标参数l与r，待查找数字x
    输出: 数字x在数组A中的下标

        else
            return BinarySearch(A,l,mid-1,x)

    现在来分析递归算法的时间复杂度:

    递归算法的时间复杂度可以分为两个部分，递归调用部分和非递归调用部分。其中，非递归调用部分的时间复杂度可以直接采用渐进分析的方法进行求解，主要需要关注的是递归部分的时间成本。

    递归的过程就是重复调用原函数解决规模更小的子问题的过程，但是在对于原问题的时间复杂度都不清楚的情况下，自然无法求解子问题部分的时间复杂度。但是，在递归原理部分提到，递归的过程存在边界，也就是在递归调用的终点，存在一个可以直接分析其复杂度的子问题。如果从反方向考虑，从递归的终点开始往前计算，那么所有子问题的时间复杂度就可以通过迭代的方式求解。每次迭代求解的流程就是将下一层子问题的时间复杂度与当前子问题非递归部分的时间复杂度相加，得到当前子问题的时间复杂度。

下面我们将介绍三种根据递推式求解时间复杂度的方法

2.3 递归树

递归树法的核心思想就是使用图示的形式描述递推表达式。递推式的计算过程一般可以被描述为一棵树，因此，对于时间复杂度的计算过程就转化为了对树上每一个节点代表的时间复杂度进行求和的过程。

2.4 代入法

代入法有一个不严谨的说法就是猜测法，就是先猜测的时间复杂度是多少，然后再用数学归纳法来证明，在证明的过程中将猜测的结果代入进去并推导是否这个结果具有一般性。这种基于猜测并代入的数学归纳法证明的方法叫做代入法
